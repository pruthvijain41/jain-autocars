This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Development
npm start                  # Start CRA dev server (localhost:3000)
npm run build              # Production build → ./build/
npm run generate-sitemap   # Regenerate public/sitemap.xml

# Firebase deployment (requires firebase-tools globally)
firebase deploy --only hosting
firebase deploy --only firestore:rules
firebase deploy --only storage

# Functions
cd functions && npm run lint   # Lint Cloud Functions before deploy
```

Environment variables must be set in `.env` (copy `.env.example`). All Firebase config keys are prefixed `REACT_APP_FIREBASE_`.

## Architecture

**Stack:** React 19 (Create React App) · React Router v7 · Tailwind CSS v3 · Framer Motion · Firebase (Auth, Firestore, Storage, Analytics) · Cloudinary (image delivery) · React Helmet (SEO)

**Routing** lives in `src/router/AppRouter.js`. Public routes use `<Layout>` (Navbar + Footer); admin routes are wrapped in both `<ProtectedRoute>` and `<AdminLayout>`.

```
Public:  /  →  HomePage
         /used-cars-in-mysore  →  BrowseCarsPage
         /car/:carId           →  CarDetailPage
         /compare              →  ComparePage
         /favorites            →  FavoritesPage
         /contact              →  ContactUsPage

Admin:   /admin/login          →  AdminLoginPage  (unprotected)
         /admin                →  AdminDashboardPage    (ProtectedRoute)
         /admin/cars           →  AdminCarsPage         (ProtectedRoute)
         /admin/inquiries      →  AdminInquiriesPage    (ProtectedRoute)
         /admin/test-drives    →  AdminTestDrivesPage   (ProtectedRoute)
         /admin/testimonials   →  AdminTestimonialsPage (ProtectedRoute)
         /admin/settings       →  AdminSettingsPage     (ProtectedRoute)
```

**Firebase access** is centralised in `src/firebase/firebase.js`, which exports `app`, `auth`, `db`, `storage`, `analytics`. Import from there — never initialise Firebase elsewhere.

**Firestore collections:**
- `cars` — public read, auth-required write
- `inquiries` — public create, auth-required read/update/delete
- `testimonials` — public create; read only if `approved == true`; auth-required update/delete
- `testDrives` — public create, auth-required read/update/delete
- `showroomInfo` — public read, auth-required write

**Image uploads** go through `src/components/forms/ImageUpload.js`, which handles both Firebase Storage and Cloudinary.

**Tailwind theme** (`tailwind.config.js`) defines two layers of tokens — the editorial system is the canonical one for new work:

- **Editorial (canonical):** `ivory` (warm cream — DEFAULT / soft / deep), `ink` (near-black — DEFAULT / soft / muted / faint), `champagne` (warm gold — DEFAULT / deep / light)
- **Legacy (do not extend):** `primary` (blue), `accent` (amber), `surface`, `text` — kept only for any pages not yet migrated
- **Fonts:** `font-display` / `font-serif` → Instrument Serif (editorial display), `font-sans` → Inter / Geist, `font-mono` → Geist Mono (eyebrows / labels), `font-heading` → Outfit (legacy)
- **Shadows:** `shadow-editorial`, `shadow-soft`, `shadow-card`, `shadow-card-hover`

Use the editorial tokens for new work, not raw Tailwind utility colors.

**Buyer-side persistence** (favorites + compare bin) lives in `src/utils/favorites.js`. It uses `localStorage` and broadcasts a custom `jain:storage` event on every write so cross-component listeners refresh.

**SEO** is handled per-page via `react-helmet`. The sitemap at `public/sitemap.xml` is generated by `generate-sitemap.js` and must be regenerated manually when routes change.

**Firebase Hosting** serves the `build/` directory. `/browse` permanently redirects to `/used-cars-in-mysore`. All paths rewrite to `index.html` (SPA mode).

**Cloud Functions** (`functions/`) are scaffolded but empty. Deploy requires passing the ESLint lint step (`functions/package.json`).
